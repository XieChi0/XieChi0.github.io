这里你看到的"终端语言"，在 WSL / Ubuntu 里主要叫：

**Shell 命令**，更具体一点通常是 **Bash 命令**。

它不是 JavaScript、Python 那种主要用来写业务程序的语言，而是用来"操作系统"的命令语言，比如：进入目录、查看文件、创建文件、删除文件、启动程序、配置环境等。

````md
# Linux / Bash 终端基础入门

## 1. 这到底是什么语言？

在 WSL / Ubuntu / Linux 终端里，我们平时输入的这些内容一般叫：

- Shell 命令
- Bash 命令
- Linux 命令
- 命令行指令

其中：

- **Shell**：命令解释器，负责理解你输入的命令
- **Bash**：Shell 的一种，Ubuntu 默认常见
- **Terminal / 终端**：你输入命令的窗口
- **Command / 命令**：你真正输入执行的内容

可以这样理解：

```txt
终端窗口
  ↓
Bash 解释你输入的命令
  ↓
操作系统执行命令
````

---

## 2. 终端提示符怎么看？

比如你看到：

```bash
zjy@DESKTOP-FGI9TOM:~/.hermes$
```

它大概由几部分组成：

```bash
用户名@电脑名:当前目录$
```

解释：

```bash
zjy                 # 当前用户
DESKTOP-FGI9TOM     # 当前电脑名
~/.hermes           # 当前所在目录
$                   # 普通用户提示符
```

如果最后是：

```bash
#
```

通常代表当前是 root 管理员权限，要更小心。

## 3. 路径的基本概念

### 3.1 当前目录

查看当前自己在哪个目录：

```bash
pwd
```

意思是：

```txt
print working directory
打印当前工作目录
```

---

### 3.2 查看目录内容

```bash
ls
```

查看详细信息：

```bash
ls -l
```

查看隐藏文件：

```bash
ls -a
```

两个一起用：

```bash
ls -la
```

---

### 3.3 进入目录

```bash
cd 目录名
```

比如：

```bash
cd ~/.hermes
```

进入上一级目录：

```bash
cd ..
```

回到用户主目录：

```bash
cd ~
```

或者：

```bash
cd
```

## 4. WSL 里的 Windows 磁盘路径

你在 Windows 里有 C 盘、D 盘、F 盘。

在 WSL 里，它们通常会挂载成：

```bash
/mnt/c
/mnt/d
/mnt/f
```

比如 Windows 里的：

```txt
F:\wuchuan_yunshui
```

在 WSL 里通常是：

```bash
/mnt/f/wuchuan_yunshui
```

所以你看到：

```bash
/mnt/f/wuchuan_yunshui/
```

意思就是：

```txt
Windows 的 F 盘里的 wuchuan_yunshui 文件夹
```

## 5. 常见文件操作命令

### 5.1 创建文件

```bash
touch test.txt
```

意思是创建一个空文件。

如果文件已经存在，`touch` 会更新它的修改时间。

---

### 5.2 创建文件夹

```bash
mkdir my-folder
```

创建多级目录：

```bash
mkdir -p a/b/c
```

---

### 5.3 删除文件

```bash
rm test.txt
```

注意：`rm` 是删除命令，比较危险。

---

### 5.4 删除文件夹

删除空文件夹：

```bash
rmdir my-folder
```

删除非空文件夹：

```bash
rm -r my-folder
```

强制删除：

```bash
rm -rf my-folder
```

非常危险的命令：

```bash
rm -rf /
rm -rf *
rm -rf /mnt/f/*
```

看到这类命令一定要非常小心。

---

### 5.5 复制文件

```bash
cp a.txt b.txt
```

复制文件夹：

```bash
cp -r folder1 folder2
```

---

### 5.6 移动 / 重命名

重命名文件：

```bash
mv old.txt new.txt
```

移动文件：

```bash
mv test.txt ./backup/
```

## 6. 查看文件内容

### 6.1 直接输出文件内容

```bash
cat file.txt
```

适合看短文件。

---

### 6.2 分页查看文件

```bash
less file.txt
```

常用操作：


### 6.3 查看前几行

```bash
tail -f app.log
```

## 8. 命令组合符号

### 8.1 `&&`

```bash
touch test.txt && rm test.txt
```

意思是：

```txt
先执行 touch test.txt
如果成功，再执行 rm test.txt
```

你在 Hermes 里看到的命令类似：

```bash
touch /mnt/f/wuchuan_yunshui/.hermes_test_write && rm /mnt/f/wuchuan_yunshui/.hermes_test_write
```

意思是：

```txt
先创建一个测试文件
创建成功后再删除它
用来测试这个目录有没有写入权限
```

---

### 8.2 `|`

管道，把前一个命令的结果交给后一个命令处理。

```bash
ls | grep test
```

意思是：

```txt
先列出文件
再从里面筛选包含 test 的内容
```

---

### 8.3 `>`

把输出写入文件，会覆盖原内容。

```bash
echo hello > a.txt
```

---

### 8.4 `>>`

把输出追加到文件末尾，不覆盖原内容。

```bash
echo hello >> a.txt
```

## 9. 搜索内容

### 9.1 grep

```bash
grep "keyword" file.txt
```

在文件里搜索 keyword。

忽略大小写：

```bash
grep -i "keyword" file.txt
```

递归搜索当前目录：

```bash
grep -r "keyword" .
```

### 9.2 find

查找文件：

```bash
find . -name "*.js"
```

意思是：

```txt
从当前目录开始，查找所有 .js 文件
```

## 10. 权限相关

### 10.1 查看权限

```bash
ls -l
```

你可能会看到：

```bash
-rw-r--r-- 1 zjy zjy 123 May 23 test.txt
```

前面的：

```txt
rw-r--r--
```

表示权限。

常见含义：

```txt
r = read，读
w = write，写
x = execute，执行
```

---

### 10.2 添加执行权限

有些脚本需要执行权限：

```bash
chmod +x install.sh
```

然后才能运行：

```bash
./install.sh
```

---

### 10.3 sudo

```bash
sudo 命令
```

意思是用管理员权限执行命令。

比如：

```bash
sudo apt update
```

注意：看到 `sudo` 要谨慎，因为它权限更高。

## 15. .env 文件

很多工具会用 `.env` 文件保存配置。

比如：

```bash
~/.hermes/.env
```

`.env` 是文件，不是目录，所以不能：

```bash
cd .env
```

这会报错：

```bash
Not a directory
```

查看 `.env` 里有没有某个配置：

```bash
grep "OPENAI_BASE_URL" ~/.hermes/.env
```

不要直接：

```bash
cat ~/.hermes/.env
```

因为里面可能有 API Key。

## 19. Hermes 的 Dangerous Command 是什么意思？

Hermes 可能会帮你执行系统命令。

如果命令里包含删除、修改权限、移动文件等危险操作，它会提示：

```txt
Dangerous Command
```

常见原因：

```bash
rm
sudo
chmod
mv
curl | bash
```

这不一定代表它一定有问题，只是提醒你检查。

建议流程：

```txt
1. 先选 Show full command
2. 看完整命令
3. 确认只是测试或安全操作，再 Allow once
4. 不要轻易 Add to permanent allowlist
```

## 20. 新手判断危险命令的方法

比较安全的命令：

```bash
pwd
ls
cd
mkdir
touch 临时文件
cat
grep
head
tail
```

需要小心的命令：

```bash
rm
mv
chmod
sudo
curl
wget
bash install.sh
```

非常危险的命令：

```bash
rm -rf /
rm -rf *
rm -rf /mnt/c/*
rm -rf /mnt/f/*
sudo rm -rf ...
```

看到这类命令，不要直接允许。
