## 冒号的作用：有时候是表示类型注解，有时候是表分隔（前面是参数类型，后面是返回类型），要怎么区分呢？

事实上，**冒号**只有一个作用，就是**给左边标注类型，即类型注解**

1. **第一种：冒号左边是「参数 / 变量名」**

作用：给这个参数 /变量 标注类型

```
// 冒号左边是 path → 给 path 标类型
path: string
```

翻译： path 这个变量，类型是  string 



2. **第二种：冒号左边是「函数的右括号 )」**

作用：给整个函数 标注「返回值类型」

```
// 冒号左边是 ) → 给函数标返回类型
): string[]
```

翻译：这个函数执行完，**返回值**的类型是  string[]

``````
function parseCategoryPath(path: string): string[] {
    //                      ↑            ↑
    //                冒号左边是path   冒号左边是)
    //             标注参数类型      标注返回值类型
}
``````

## ts里的<>怎么理解呢

👉 **泛型 = 类型的“参数机制”**

- 定义时：是“占位符”
- 使用时：是“传入具体类型”

> * 定义时 类型占位符
>
> ``````
> function fn<T>(value: T): T {
>   return value
> }
> ``````
>
> * 使用时 传入类型
>
> ``````
> fn<number>(123)
> ``````
>
>  
>
> > ``````
> > function ref<T>(value: T): Ref<T>
> > ``````
> >
> > 使用时：```ref<numver>(0)```
> >
> > 如果不用泛型时，会出现以下情况
> >
> > const user =ref(null)
> >
> > ts会推断为ref<null>
> >
> > 那么之后写user.value={name:"a"}会报错
> >
> > 正确的写法是
> >
> > type User ={name:string}
> >
> > const user =ref<User|null>(null)

## 那ts只能通过<>来传参吗？

首先问题有误，<>是传类型的 ```fn<number>(123)```

不过这里的<>可以不写，因为ts会自动推断，根据你传参的类型推断类型

但如果你担心ts推断错误，所以还是写上<类型>



下面讲讲其他定义类型机制

1️⃣声明

```let a:number=1```

直接指定类型

2️⃣as断言

```let a ="123" as string```

强行告诉ts这是string

3️⃣泛型（这个本质上是传类型而不是定义类型）

``````
Array<number>
ref<User>()
``````



## 断言可以强行把一个类型说成另一个类型吗

不可以直接这样写

```let a  = "123" as boolean```

ts不会允许

但是可以

```let a = "123" as unknown as boolean```

因为断言的本质是改变ts的类型判断，但不改变真实值，所以说即使判断变了，也许会影响if(a)的判断，但是本质上它还是字符串，数据不变。

# 为什么括号代表函数呢

首先这个说法是错误的，因为在js/ts中，只要是函数，就必须带一对（）

这是**语法规则**，就像：

- 句子结尾要句号

- 代码块要用  { } 

<h3>小括号  () 在函数里的专属作用</h3>

**1. 定义函数时**

```
function 函数名( 放参数 ) : 返回类型 {
}
```

-  () 唯一作用：专门用来装参数

- 哪怕没有参数，括号也不能省

```
// 无参函数，括号必须留着
function getTime(): number {
  return Date.now()
}
```



**2. 调用函数时**

```
parseCategoryPath("分类/路径")
```

 () 代表：**执行这个函数**
