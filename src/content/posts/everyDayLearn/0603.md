# 立即执行函数（IIFE）与闭包

> 本文梳理 JavaScript 中 IIFE 和闭包的关系，两者常常一起使用，理解它们的联系很重要。

## 什么是闭包？

**闭包（Closure）** 是指一个函数能够"记住"并访问它词法作用域外部的变量，即使这个函数在它定义的词法作用域之外执行。

简单说：**函数 + 它能访问的外部变量 = 闭包**

```javascript
function outer() {
  const a = 1
  function inner() {
    console.log(a)  // inner 能访问 outer 的变量 a
  }
  return inner
}

const fn = outer()  // fn 就是一个闭包
fn()  // 输出 1，即使 outer() 已经执行完了，a 仍然被"记住"了
```

## 立即执行函数（IIFE）

IIFE（Immediately Invoked Function Expression），即立即调用函数表达式，定义的同时就执行这个函数，执行完就销毁，不会污染外部作用域。

### 变成立即执行函数的两种经典写法

```javascript
// 写法1：用()把整个函数表达式包起来
(function() {
  console.log('我立即执行了！')
})()

// 写法2：用()把函数包起来，后面再加()
(function() {
  console.log('我也立即执行了！')
}())
```

### IIFE 的经典应用：循环中的闭包问题

```javascript
// 普通写法：i 会变成全局变量，污染环境
for (var i = 1; i <= 3; i++) {
  setTimeout(() => console.log(i), i * 1000)
}
// 输出 4 4 4（经典闭包问题）

// 用 IIFE 解决：每个循环生成独立作用域，保存当前 i
for (var i = 1; i <= 3; i++) {
  (function(j) {
    setTimeout(() => console.log(j), j * 1000)
  })(i)
}
// 输出 1 2 3 ✅
```

## IIFE 和闭包的关系

**IIFE 本身就是闭包的一种应用场景。**

### 闭包是什么？谁是谁？

先搞清楚这两个"谁"：

- **外层函数**：`function(j) { ... }` —— 就是 IIFE 本身
- **内部函数**：setTimeout 里的 `() => console.log(j)` —— 箭头函数

箭头函数定义在 IIFE 内部，所以它的外层函数就是 IIFE，而它引用了 IIFE 的局部变量 `j`，这就构成了闭包。

### 分步拆解（以第一轮 i=1 为例）

```javascript
(function(j) {        // ← IIFE 执行，实参 i=1 传给形参 j，此时 j = 1
  setTimeout(() => console.log(j), j * 1000)  // ← 箭头函数引用了 j
})(i)  // ← IIFE 执行完毕
```

1. **IIFE 执行**，全局 `i=1` 传入，IIFE 内部生成局部变量 `j = 1`
2. **在 IIFE 内部创建箭头函数**，作为 setTimeout 回调
3. **箭头函数引用了 `j`** —— j 是 IIFE 的局部变量
4. **IIFE 执行完毕** —— 按理说函数执行完，局部变量应该被销毁
5. **但定时器还活着**，回调还持有对 `j` 的引用
6. → JS 引擎不会回收 `j`，j 被保留了下来

> 这种「内部函数引用了外层函数的局部变量」的现象，就是闭包。

### 三个循环轮次各有一个独立的闭包

| 循环轮次 | IIFE 内的 j | 定时器回调 |
| --- | --- | --- |
| 第1轮 | j = 1 | 记住 j=1，1秒后输出 1 |
| 第2轮 | j = 2 | 记住 j=2，2秒后输出 2 |
| 第3轮 | j = 3 | 记住 j=3，3秒后输出 3 |

**每个 IIFE 都创建了独立的作用域**，每个作用域里的 `j` 互不影响，所以输出 1 2 3。

IIFE 解决闭包问题的本质：**通过创建新作用域，让每个闭包记住不同的值**。

## 闭包的常见应用场景

### 1. 数据私有化（模块化）

```javascript
function createCounter() {
  let count = 0  // 私有变量，外部无法直接访问
  return {
    increment() { count++ },
    decrement() { count-- },
    getCount() { return count }
  }
}

const counter = createCounter()
counter.increment()
counter.increment()
console.log(counter.getCount())  // 2
console.log(counter.count)  // undefined，外部无法访问
```

### 2. 函数工厂（缓存/记忆化）

```javascript
function multiplicationFactory(base) {
  return function(num) {
    return base * num
  }
}

const double = multiplicationFactory(2)
const triple = multiplicationFactory(3)

console.log(double(5))   // 10
console.log(triple(5))   // 15
console.log(double(10))  // 20
```

### 3. 延迟执行 / 回调

```javascript
// React 中的经典用法
function fetchData(id) {
  return fetch(`/api/${id}`)
    .then(res => res.json())
}

// 事件监听
button.addEventListener('click', function() {
  console.log('点击了', id)  // 闭包引用了外部的 id
})
```

## 闭包的注意事项

### 1. 内存泄漏风险

如果闭包长期持有大对象引用，可能导致内存无法释放：

```javascript
function heavy() {
  const bigArray = new Array(1000000)
  return function() {
    console.log(bigArray.length)  // bigArray 一直被引用，无法被 GC
  }
}

const fn = heavy()
// 使用完记得手动解除引用
fn = null  // 或者将 fn = undefined
```

### 2. this 指向问题

闭包中的 `this` 指向需要注意：

```javascript
const obj = {
  name: 'obj',
  getName() {
    return function() {
      console.log(this.name)  // this 指向 window，不是 obj
    }
  }
}

// 解决：用箭头函数或者缓存 this
const obj2 = {
  name: 'obj',
  getName() {
    const self = this
    return function() {
      console.log(self.name)  // 用 self 保存 this
    }
  }
}
```

## 总结

| 概念 | 说明 |
| --- | --- |
| 闭包 | 函数 + 记住外部变量 |
| IIFE | 立即执行 + 创建独立作用域 |
| 关系 | IIFE 是利用闭包解决变量问题的经典手段 |

**核心理解**：闭包让函数拥有"记忆能力"，IIFE 则利用这个能力创建独立作用域，避免变量污染和共享问题。
