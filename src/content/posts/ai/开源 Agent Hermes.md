## Hermes 的真正核心

Hermes 之所以火，不是因为它用了什么更强的模型。

它一直在提的核心卖点是：**"The agent that grows with you"——会和你一起成长**。

这意味着 Hermes 不是个工具，它是个学生。你用它完成一件事，它就会记住你是如何完成的。下次再做类似的事情时，它就会直接按照你的方式来做，不用你再重复交代。

传统 AI：你关掉窗口它就忘了。Hermes 会记忆。

真正的核心差异——**不是模型，而是学习的方式**。

---

## 核心机制：Self-Improving Loop

Hermes 最核心的东西——**Self-Improving Loop（自我改进循环）**，也是它跟所有其他 Agent 方案的实质性不同。

这是一个**三层递进的循环**：

| 层级 | 功能 | 说明 |
|------|------|------|
| 第一层 | 记忆积累 | 用户的偏好、项目背景、工作风格、常用工具习惯等 |
| 第二层 | 技能生成 | 把完成任务的流程提炼成技能文档（skill.md） |
| 第三层 | 持续进化 | 用户用得越久，积累越多，效率越高 |

循环往复，**越用越聪明**。

---

## 第一层：持久记忆系统

第一层是记忆积累，包括用户的偏好、项目背景、工作风格、常用工具习惯等。Hermes 会跨会话式地全部记住，用户不用每次都重新解释。

### 底层技术

持久记忆系统底层用的是 **SQLite + FTS5 支持全文检索**。

上下文窗口是有限的，不能无限制往里面塞东西。所以 Hermes 有一层 **LLM Summarization**——自动压缩历史对话，把长对话里最关键的信息提炼出来，用更少的 token 表达同样的信息量。

还有一个 **渐进式建立用户画像（Dialectic User Profiling）的机制**——不是一次性记住你所有信息，而是在多次交互中慢慢拼凑出"你是谁、你关心什么、你喜欢什么样的工作方式"。

---

## 第二层：自动技能创建

很多 Agent 的技能需要你手动去安装、去配置，Open Cloud 就是这样：你需要什么技能去它的插件市场里面找，安装好配置好才能用。

**Hermes 不一样，它是自动的。**

当你让 Hermes 完成一个复杂的任务，比如"帮我从 0~1 搭建个人网站"，它在完成之后会自动生成一个 `skill.md` 文件记录：

- 这次是怎么搭的
- 有哪些关键步骤
- 下次遇到类似的应该怎么去做

这个文件是**永久存储**的。你下次再让它做个人网站，它会先看看有没有已经存在的相关技能，有的话直接调用，没有的话才会重新规划。

这个技能格式**兼容 Agent Skill 开放标准**，意味着你写的技能可以被其他人用，也可以使用其他人分享的技能。

这就是把"用完就忘"变成"越用越强"的机制。

---

## 第三层：持续精炼

这里有个关键问题：Hermes 用得越久，自动生成的 Skill 会越积越多，可能导致效率下降。

所以第三层是**持续精炼**——把前两层能力闭环跑通，自动优化。这也是 Hermes 最核心的自我进化引擎。

首先是把**重复或者相近的 Skill 自动融合**，避免技能库拥挤。**工具不是越多越好，精简才能保持高效。**

每次用技能完成任务，Hermes 会复盘更快更省 token 的路径，并且直接更新旧的技能来让它越跑越顺畅。在记忆里也会不断提炼你的工作风格，**自动预判你的需求**，而不是等待你继续下达指令。

## Hermes vs OpenClou

Open Cloud（圈内俗称"龙虾"）是 Hermes 之前最火的开源 Agent，36.2 万星，生态成熟，插件市场超过数千个技能平台，覆盖 25 个渠道。

很多人讨论这两个谁更好用。但这应该不是一个"谁比谁强"的问题，因为**他们的设计理念就不同**。

### 设计理念对比

| 维度 | Open Cloud | Hermes |
|------|------------|--------|
| 核心定位 | 人在决策链的中心 | Agent 是进化主角 |
| 交互方式 | 你告诉它做什么 | 它自己学会你的方式 |
| Gateway | 管理 session、工具、事件的 WebSocket | 你让它做的事越多，它越懂 |

### 四维度具体对照

#### 1. 记忆机制

- **Open Cloud**：纯 Markdown 文件存储 memory 和 user，需要手动编辑，不会自动整理
- **Hermes**：全自动。你跟它交互的过程中，它会自动压缩、自动提炼、自动建立用户画像

#### 2. 技能获取

- **Open Cloud**：手动去插件市场搜、安装、调试
- **Hermes**：自动生成。你每让它做一个任务，它自己把这次经验提炼成技能文档

#### 3. 部署成本

- **Hermes**：支持 Docker、SSH、Serverless 多种方式，5 美元的 VPS 就能跑起来
- **Open Cloud**：主要本地运行，Serverless 部署需要自己搞

#### 4. 安全模型

- **Hermes**：沙盒隔离、容器执行、零遥测，不收集不上传任何使用数据
- **Open Cloud**：DM Ping 验证加显示 Allow List，每条消息都需要验证，敏感操作需要审批

### 适用场景

| 场景 | Hermes | Open Cloud |
|------|--------|------------|
| 信任 Agent 自主性的个人用户 | ✅ | ❌ |
| 预算敏感，想低成本部署 | ✅ | ❌ |
| 不想折腾配置 | ✅ | ❌ |
| 企业级部署，需要完整审计和合规 | ❌ | ✅ |
| 需要精细控制每一条指令 | ❌ | ✅ |
| Agent 每个动作都需要审批 | ❌ | ✅ |

### 总结

> **Open Cloud**：你告诉它做什么
> **Hermes**：它学会你自己想做什么

这是一个"你想让 AI 以什么方式陪伴你"的问题。

## 技术架构

Hermes Agent 的架构分为四层：

### 最外层：用户交互层

支持 CLI、Telegram、Discord、Slack、WhatsApp、Signal 等消息平台。有消息网关进程统一管理所有平台的接入。

### 中间层：核心智能体引擎

包括记忆系统、技能系统、工具执行和规划推理四个模块。这四个模块配合在一起才能实现 Self-Improving Loop。

官方说法是："Every 15 tool calls is a self-evaluation checkpoint"——每 15 次工具调用就命中自我进化的 checkpoint，然后开始创建技能。

### 执行环境层

支持本地终端、Docker、SSH、远程、Model Singularity 五种运行方式。

### 最底层：大模型支持层

支持 OpenRouter、自定义 API、本地 VLLM，共 200 多个模型可供选择，一条命令就能切换。

Hermes **不想被任何一个模型厂商绑定**。

### 数据存储与安装

- **数据存储**：系统数据存在本地 `~/.hermes` 目录，完全自托管，不在云端，无遥测数据
- **安装方式**：一条 curl 命令，不需要 Root 权限，支持 Linux、MacOS、WSL2
