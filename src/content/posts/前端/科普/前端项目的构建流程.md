# 前端项目的构建流程

## 一、核心概念区分：编译 vs 构建 vs 运行

这三个概念经常被混用，但实际上有明确的区分：

### 1. 编译（Compile）— 代码翻译

**定义**：将高级语言/框架语法转译成浏览器能理解的形式。

| 编译做的事 | 例子 |
|-----------|------|
| 语言转译 | TypeScript → JavaScript |
| 框架语法转译 | `.svelte` → 原生 JS，JSX → `createElement` |
| 内容转译 | Markdown → HTML |
| 模板转译 | Vue `<template>` → 渲染函数 |

**核心特点**：代码翻译，**不涉及打包和优化**。

### 2. 构建（Build）— 编译 + 打包优化

**定义**：构建 = 编译 + 打包 + 压缩 + 优化 + 输出。

```
构建（build）
├── 编译（compile）    ← 代码翻译
├── 打包（bundle）     ← 合并文件（多个 JS/CSS → 少数几个）
├── 压缩（minify）     ← 删除空白、缩短变量名
├── 优化（tree-shaking）← 删除未使用代码
└── 输出产物（dist/） ← 最终静态文件
```

**关键理解**：构建 > 编译，打包只是构建的一部分。
在开发环境，不会进行完整构建，只是按需编译 并通过本地开发服务器（dev server）提供给浏览器。
当发布到生产环境时，才会执行完整的构建，生成优化后的静态资源。


### 3. 运行时（Runtime）— 浏览器执行

**定义**：代码在用户浏览器中执行，产生交互和界面变化。

| 运行时做的事 | 例子 |
|-------------|------|
| 下载文件 | 浏览器请求 HTML/CSS/JS |
| 解析代码 | 浏览器解析 HTML/CSS/JS |
| 执行 JS | 点击按钮触发事件处理函数 |
| 访问 API | 搜索时请求 pagefind |
| 更新界面 | 点击后数字 +1 |

## 三、编译 vs 构建 vs 运行 详细对比

| 阶段 | 做什么 | 发生在哪里 | 开发时 | 构建时 |
|------|--------|-----------|--------|--------|
| **编译** | 语言/框架语法转译 | 你的电脑 / CI 服务器 | ✅ 按需编译 | ✅ 全部编译 |
| **构建** | 编译 + 打包 + 压缩 + 优化 | 你的电脑 / CI 服务器 | ❌ 不完整 | ✅ 完整执行 |
| **运行** | 下载文件、解析代码、执行 JS | 用户的浏览器 | ✅ 执行 | ✅ 执行 |

### 一句话总结

> **编译** = 翻译代码（.svelte → JS），**构建** = 编译 + 打包 + 优化 + 输出，**运行** = 在浏览器中执行。

## 五、pnpm dev 到底发生了什么？

`npm run dev` 不是单纯的"运行时"，它是一个**混合模式**——同时做编译和服务：

```
┌─────────────────────────────────────────────────────────────┐
│                    pnpm dev 启动后                            │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   开发服务器 (Dev Server)                                    │
│                                                             │
│   ┌───────────┐    ┌───────────┐    ┌───────────┐          │
│   │  启动服务器 │ ──▶ │  监听文件 │ ──▶ │  请求到来  │          │
│   │  localhost │    │  变化     │    │  实时编译  │          │
│   │  :4321    │    │           │    │  返回页面  │          │
│   └───────────┘    └───────────┘    └───────────┘          │
│                                                             │
│   你改代码 → 保存 → 浏览器自动刷新 → 新的页面                 │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 详细流程

**1. 你输入 `pnpm dev`**

```
┌─────────────────────────────────────────────────────────────┐
│                    开发服务器启动                            │
│                                                             │
│   1. 启动本地服务器 (http://localhost:4321)                 │
│   2. 初始化编译器（Astro 内置的 Vite）                       │
│   3. 监听 src/ 目录下所有文件变化                           │
└─────────────────────────────────────────────────────────────┘
```

**2. 你打开浏览器访问 http://localhost:4321/**

```
┌─────────────────────────────────────────────────────────────┐
│                    第一次请求页面                            │
│                                                             │
│   ┌─────────────────────────────────────────────────────┐  │
│   │  Vite/Astro 实时编译                                  │  │
│   │                                                     │  │
│   │  src/pages/index.astro  ──▶  HTML                   │  │
│   │  src/components/...         ──▶  HTML               │  │
│   │  src/config/...             ──▶  数据                │  │
│   └─────────────────────────────────────────────────────┘  │
│   返回编译后的 HTML 给浏览器                                 │
└─────────────────────────────────────────────────────────────┘
```

**3. 你修改代码并保存**

```
┌─────────────────────────────────────────────────────────────┐
│                    文件变化检测                              │
│                                                             │
│   Vite 检测到文件变化                                        │
│        │                                                   │
│        ▼                                                   │
│   只重新编译变化的文件及其依赖                               │
│   其他没变的页面保留缓存                                     │
│        │                                                   │
│        ▼                                                   │
│   浏览器自动刷新（Hot Module Replacement）                   │
└─────────────────────────────────────────────────────────────┘
```

## 六、开发环境 vs 生产环境 对比

### 开发时 (dev)

```
┌─────────────────────────────────────────────────────────┐
│                                                         │
│   编译器状态：待机 ←── 收到请求 ──→ 编译 ──→ 返回结果    │
│                                                         │
│   特点：                                                │
│   ├── 不生成 dist/ 文件                                 │
│   ├── 编译结果在内存中                                  │
│   ├── 页面按需编译（访问时才编译）                      │
│   └── 修改后自动重新编译                                │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 生产构建 (build)

```
┌─────────────────────────────────────────────────────────┐
│                                                         │
│   编译器状态：全力编译 ──→ 所有页面 ──→ 输出 dist/       │
│                                                         │
│   特点：                                                │
│   ├── 生成真正的文件到 dist/                            │
│   ├── 编译所有页面（不管你访问没访问）                  │
│   ├── 代码压缩、优化                                    │
│   ├── 代码分割（按需加载）                              │
│   └── 部署到服务器后可删除源代码                        │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

## 八、关键结论

- **`pnpm dev`** 是"边编译边运行"的混合模式——既不是纯粹的编译时，也不是纯粹的运行时
- **编译** 是构建的一部分，构建 = 编译 + 打包 + 优化 + 输出
- **开发环境** 和 **生产环境** 都会经历"编译"和"运行"，但"构建"通常只在生产阶段完整执行
- **编译时** = 你的电脑/CI 服务器处理代码，**运行时** = 用户浏览器执行代码
