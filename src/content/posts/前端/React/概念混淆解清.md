## 状态的含义

这个问题很好！让我帮你澄清 **"状态"（State）** 的确切含义。

## React 中的"状态"是特定术语

**"状态"在 React 中有特指含义**：只有通过 `useState` 创建的数据才叫"状态"。

**不是所有变量都是"状态"**

```jsx
function App() {
  // ✅ 这是状态（useState 创建的）
  const [count, setCount] = useState(0)

  // ❌ 这是普通变量（不是状态）
  const name = "Alice"

  // ❌ 这也是普通变量（不是状态）
  const result = count * 2

  // ❌ 这还是普通变量（不是状态）
  const stats = { total: 10, active: 5 }
}
```

**重新渲染 ≠ 状态改变**

| 术语                         | 含义                          |
| ---------------------------- | ----------------------------- |
| **渲染（Render）**           | 组件函数重新执行一次          |
| **状态改变（State Change）** | `useState` 的值从旧值变成新值 |

```jsx
function Counter() {
  const [count, setCount] = useState(0)  // 初始值 0

  console.log("渲染了")  // 每次渲染都会打印

  return (
    <button onClick={() => setCount(count + 1)}>
      {count}
    </button>
  )
}
```

运行流程：
1. 首次渲染 → `count = 0`，打印 "渲染了"
2. 点击按钮 → `setCount(1)` → `count` 变成 `1` → **状态改变**
3. 重新渲染 → `count = 1`，打印 "渲染了"
4. 点击按钮 → `setCount(2)` → `count` 变成 `2` → **状态改变**
5. 重新渲染 → `count = 2`，打印 "渲染了"

**对照表**

| 概念     | React                 | Vue                           |
| -------- | --------------------- | ----------------------------- |
| 状态     | `useState` 创建的变量 | `ref` / `reactive` 创建的变量 |
| 改变状态 | `setXxx()`            | `.value = xxx`                |
| 响应式   | 状态变化 → 重新渲染   | 数据变化 → 自动响应           |
| 普通变量 | 每次渲染重新计算      | 同上                          |

**你的代码中的例子**

```jsx
// useTodoList.ts
const [todos, setTodos] = useState([])  // ✅ 状态
const [filter, setFilter] = useState('all')  // ✅ 状态
const stats = { ... }  // ❌ 普通变量，不是状态
```

**总结**：
- **状态** = `useState` 创建的变量，只有这些变化才会触发重新渲染
- **普通变量** = 每次渲染重新计算，不会自动触发更新
- **重新渲染** ≠ 状态改变，但状态改变一定会重新渲染

## 懒加载vs函数式更新

两者虽然都用到了箭头函数，但目的不同：
- **懒加载** = 优化初始化，避免重复计算
- **函数式更新** = 基于最新状态计算新值，避免状态依赖错误

`useState` 中的懒加载（Lazy Initialization）和函数式更新（Functional Update）确实容易混淆，它们都与"函数"有关，但用途完全不同：

### 1. 懒加载（Lazy Initialization）

```
const [todos, setTodos] = useState<Todo[]>(() => loadFromStorage(defaultTodos))
```

**为什么需要懒加载？**

| 写法 | 效果 |
|------|------|
| `useState(loadFromStorage(defaultTodos))` | 组件**每次渲染**都会执行 `loadFromStorage` |
| `useState(() => loadFromStorage(defaultTodos))` | 只在**首次渲染**时执行一次 |

React 只会在初始化阶段调用这个箭头函数，之后就"记住"返回值了，不会再调用它。

**典型场景**：
- 读取 localStorage（稍微耗时）
- 计算量较大的初始化逻辑
- 需要遍历大量数据

### 2. 函数式更新（Functional Update）

```
setTodos(prev => [newTodo, ...prev])
```

**为什么需要函数式更新？**

`prev` 参数代表**当前最新状态**。这在以下场景很有用：

- **新状态依赖旧状态**（如往数组头部添加元素）
- **批量更新时避免竞态**（连续快速调用时，确保基于最新状态）

### 总结对比

| 概念 | 位置 | 传入函数的形式 | 触发时机 |
|------|------|---------------|---------|
| 懒加载 | `useState(initialFunc)` | `() => value` | 首次渲染 |
| 函数式更新 | `setState(updateFunc)` | `prev => newValue` | 每次更新 |

## 渲染、组件、Hook 的关系

### 1. 组件和 Hook 的区别

| 类型        | 例子              | 作用                          |
| ----------- | ----------------- | ----------------------------- |
| 组件        | `App`、`TodoList` | 返回 JSX，描述页面            |
| 自定义 Hook | `useTodoList`     | 封装状态和逻辑，不直接返回 UI |

`useTodoList` 不是组件，它只是一个被组件调用的函数。只是因为它内部用了 React Hook，所以名字必须以 `use` 开头，并且只能在组件或其他 Hook 里调用。

### 2. 重新渲染时发生了什么

```tsx
function App() {
  const { todos, addTodo } = useTodoList()

  return (
    <div>
      <TodoInput onAdd={addTodo} />
      <TodoList todos={todos} />
    </div>
  )
}
```

当 `App` 重新渲染时：

1. `App` 函数重新执行。
2. `useTodoList()` 也会再次被调用。
3. `useTodoList` 里面的 `useState`、`useCallback`、普通计算都会再次执行。
4. React 根据 Hook 调用顺序，把上一次保存的 state 和缓存值对应回来。

所以要区分两件事：

- Hook 函数体会再次执行。
- `useState` 保存的状态不会丢失。

### 3. 常见 Hook 在重新渲染时的表现

| 代码            | 每次渲染都会执行吗 | 结果                                  |
| --------------- | ------------------ | ------------------------------------- |
| `useState()`    | 会                 | React 返回保存过的状态值              |
| `useCallback()` | 会                 | 依赖没变时返回缓存函数                |
| `useMemo()`     | 会                 | 依赖没变时返回缓存结果                |
| `useEffect()`   | 会被注册           | effect 回调在渲染后、且依赖变化时执行 |

注意：`useEffect` 的依赖变化不是“触发组件重新渲染”的原因。通常是先因为 state / props 变化导致组件重新渲染，渲染完成后 React 再检查依赖是否变化，决定要不要执行 effect。

如果 effect 里面又调用了 `setState`，那才会触发下一次渲染。

### 4. 重新渲染的常见原因

| 触发条件              | 说明                                    |
| --------------------- | --------------------------------------- |
| 组件自己的 state 更新 | 调用 `setState` 后，React 安排重新渲染  |
| 父组件重新渲染        | 默认情况下，子组件也会跟着重新渲染      |
| props 变化            | 父组件传入的新 props 可能导致子组件更新 |
| context 变化          | 使用了该 context 的组件会更新           |

### 5. 最后总结

React 初学阶段先抓住这几条就够了：

1. **重新渲染 = 组件函数重新执行。**
2. **每次渲染都是一张快照。**
3. **state 不要直接改，要创建新值。**
4. **能直接计算出来的数据，不要用 `useEffect` 存成新 state。**
5. **`useCallback` 不是默认必用，主要用于稳定传给 memo 子组件或其他 Hook 的函数引用。**

理解这些以后，React 的很多“奇怪规则”都会变得更自然。

## stats={stats}

```react
<section className="hero-card">
          <span className="eyebrow">今日计划</span>
          <h1>把今天整理得轻一点。</h1>
          <p className="hero-copy">
            一个只为个人使用设计的高级感 Todo List：为自己保留温柔。
          </p>
          <TodoStats stats={stats} />
        </section>
```

这里的< TodoStats >是子组件，那么stats={stats}，用到了jsx的写法

> 如果是vue里，会这样写`:stats="stats"`，

那么说回jsx的写法，这里的等号左边是 子组件接收时用的名字，等号右边是 当前作用域中stats变量，这里用花括号包裹是因为 插入变量时要用花括号进行包裹。
