# Linux / Bash 终端基础入门

## 1. 这到底是什么语言？

在 WSL / Ubuntu / Linux 终端里，我们平时输入的这些内容一般叫：

- Shell 命令
- Bash 命令
- Linux 命令
- 命令行指令

其中：

- **Shell**：命令解释器，负责理解你输入的命令
- **Bash**：Shell 的一种，Ubuntu 默认常见
- **Terminal / 终端**：你输入命令的窗口
- **Command / 命令**：你真正输入执行的内容

可以这样理解：

```txt
终端窗口
  ↓
Bash 解释你输入的命令
  ↓
操作系统执行命令
````

---

## 2. 终端提示符怎么看？

比如你看到：

```bash
zjy@DESKTOP-FGI9TOM:~/.hermes$
```

它大概由几部分组成：

```bash
用户名@电脑名:当前目录$
```

解释：

```bash
zjy                 # 当前用户
DESKTOP-FGI9TOM     # 当前电脑名
~/.hermes           # 当前所在目录
$                   # 普通用户提示符
```

如果最后是：

```bash
#
```

通常代表当前是 root 管理员权限，要更小心。

## 4. WSL 里的 Windows 磁盘路径

你在 Windows 里有 C 盘、D 盘、F 盘。

在 WSL 里，它们通常会挂载成：

```bash
/mnt/c
/mnt/d
/mnt/f
```

比如 Windows 里的：

```txt
F:\wuchuan_yunshui
```

在 WSL 里通常是：

```bash
/mnt/f/wuchuan_yunshui
```

所以你看到：

```bash
/mnt/f/wuchuan_yunshui/
```

意思就是：

```txt
Windows 的 F 盘里的 wuchuan_yunshui 文件夹
```

## 5. 常见文件操作命令

### 5.1 创建文件

```bash
touch test.txt
```

意思是创建一个空文件。

如果文件已经存在，`touch` 会更新它的修改时间。

---

### 5.2 创建文件夹

```bash
mkdir my-folder
```

创建多级目录：

```bash
mkdir -p a/b/c
```

---

### 5.3 删除文件

```bash
rm test.txt
```

注意：`rm` 是删除命令，比较危险。

---

### 5.4 删除文件夹

删除空文件夹：

```bash
rmdir my-folder
```

删除非空文件夹：

```bash
rm -r my-folder
```

强制删除：

```bash
rm -rf my-folder
```

非常危险的命令：

```bash
rm -rf /
rm -rf *
rm -rf /mnt/f/*
```

看到这类命令一定要非常小心。

---

### 5.5 复制文件

```bash
cp a.txt b.txt
```

复制文件夹：

```bash
cp -r folder1 folder2
```

---

### 5.6 移动 / 重命名

重命名文件：

```bash
mv old.txt new.txt
```

移动文件：

```bash
mv test.txt ./backup/
```

## 8. 命令组合符号

### 8.1 `&&`

```bash
touch test.txt && rm test.txt
```

意思是：

```txt
先执行 touch test.txt
如果成功，再执行 rm test.txt
```

你在 Hermes 里看到的命令类似：

```bash
touch /mnt/f/wuchuan_yunshui/.hermes_test_write && rm /mnt/f/wuchuan_yunshui/.hermes_test_write
```

意思是：

```txt
先创建一个测试文件
创建成功后再删除它
用来测试这个目录有没有写入权限
```

---

### 8.2 `|`

管道，把前一个命令的结果交给后一个命令处理。

```bash
ls | grep test
```

意思是：

```txt
先列出文件
再从里面筛选包含 test 的内容
```

---

### 8.3 `>`

把输出写入文件，会覆盖原内容。

```bash
echo hello > a.txt
```

---

### 8.4 `>>`

把输出追加到文件末尾，不覆盖原内容。

```bash
echo hello >> a.txt
```

## 10. 权限相关

### 10.1 查看权限

```bash
ls -l
```

你可能会看到：

```bash

```

前面的：

```txt
rw-r--r--
```

表示权限。

常见含义：

```txt
r = read，读
w = write，写
x = execute，执行
```

### 10.2 添加执行权限

有些脚本需要执行权限：

```bash

```

然后才能运行：

```bash

```

### 10.3 sudo


意思是用管理员权限执行命令。

注意：看到 `sudo` 要谨慎，因为它权限更高。

## 15. .env 文件

很多工具会用 `.env` 文件保存配置。

比如：

```bash
~/.hermes/.env
```

`.env` 是文件，不是目录，所以不能：

```bash
cd .env
```

这会报错：

```bash
Not a directory
```

查看 `.env` 里有没有某个配置：

```bash
grep "OPENAI_BASE_URL" ~/.hermes/.env
```

不要直接：

```bash
cat ~/.hermes/.env
```

因为里面可能有 API Key。

## 19. Hermes 的 Dangerous Command 是什么意思？

Hermes 可能会帮你执行系统命令。

如果命令里包含删除、修改权限、移动文件等危险操作，它会提示：

```txt
Dangerous Command
```

常见原因：

```bash
rm
sudo
chmod
mv
curl | bash
```

这不一定代表它一定有问题，只是提醒你检查。

建议流程：

```txt
1. 先选 Show full command
2. 看完整命令
3. 确认只是测试或安全操作，再 Allow once
4. 不要轻易 Add to permanent allowlist
```

## 23. WSL 路径、Hermes 目录和常见命令行区别

这一节是对前面内容的补充，主要解决几个容易混淆的问题：

- 当前提示符里的 `/mnt/c/WINDOWS/System32` 是什么？
- `~/.hermes/skills` 是什么？
- `ls -la`、`cat` 这些命令怎么理解？
- Bash、CMD、PowerShell 分别适合干什么？

---

### 23.1 当前目录是什么意思？

你在终端里看到类似：

```bash
zjy@DESKTOP-FGI9TOM:/mnt/c/WINDOWS/System32$
```

可以拆成几部分：

```txt
zjy                     当前用户
DESKTOP-FGI9TOM         当前电脑名
/mnt/c/WINDOWS/System32 当前所在目录
$                       普通用户提示符
```

其中最重要的是当前目录：

```bash
/mnt/c/WINDOWS/System32
```

在 WSL 里，Windows 的磁盘通常会被映射成：

```bash
/mnt/c
/mnt/d
/mnt/f
```

所以：

```bash
/mnt/c/WINDOWS/System32
```

大概对应 Windows 里的：

```txt
C:\WINDOWS\System32
```

这个目录是 Windows 系统目录，不适合平时放项目、安装工具、改配置。  
如果你只是操作 Hermes、Node、Git、项目代码，建议先回到自己的 WSL 用户目录：

```bash
cd ~
```

再继续执行命令。

---

### 23.2 `~/.hermes/skills` 是什么？

`~` 表示当前 WSL / Linux 用户的家目录。

所以：

```bash
~/.hermes/skills
```

可以理解成：

```txt
当前用户家目录 / .hermes / skills
```

也就是：

```txt
Hermes 在当前用户目录下存放 skills 的地方
```

这里面几个部分的意思是：

```txt
~          当前用户家目录
.hermes    Hermes 的隐藏配置目录
skills     Hermes 的技能目录
```

注意：

```bash
/mnt/c/WINDOWS/System32
```

和：

```bash
~/.hermes/skills
```

不是一个地方。

简单理解：

```txt
/mnt/c/WINDOWS/System32
    是你当前“站着”的目录

~/.hermes/skills
    是 Hermes 存放 skill 文件的目录
```

你站在哪个目录，不代表 Hermes 的文件就在哪个目录。  
就像你人在客厅，但冰箱在厨房；你可以从客厅走去厨房，也可以直接说“打开厨房的冰箱”。

---

### 23.3 `~`、`.`、`..` 和隐藏文件

命令行里常见几个路径符号：

```txt
~    当前用户家目录
.    当前目录
..   上一级目录
```

比如：

```bash
cd ~
```

表示回到当前用户家目录。

```bash
cd ..
```

表示进入上一级目录。

```bash
./install.sh
```

表示执行当前目录下的 `install.sh` 文件。

以点开头的文件或目录，一般是隐藏的，例如：

```txt
.hermes
.env
.git
.vscode
```

想看到隐藏文件，需要用：

```bash
ls -a
```

---

### 23.4 `ls -la` 是什么意思？

`ls` 是查看目录内容。

```bash
ls
```

表示列出当前目录里的普通文件和文件夹。

`-l` 表示显示详细信息：

```bash
ls -l
```

`-a` 表示显示所有文件，包括隐藏文件：

```bash
ls -a
```

所以：

```bash
ls -la
```

就是：

```txt
列出当前目录下的所有文件，包括隐藏文件，并显示详细信息
```

你可能会看到类似：

```bash
drwxr-xr-x  3 zjy zjy 4096 May 26 .hermes
-rw-r--r--  1 zjy zjy  123 May 26 README.md
```

开头这些字符可以粗略这样看：

```txt
d    目录
-    普通文件
r    read，读
w    write，写
x    execute，执行
```

新手阶段不用一次看懂每一列，先记住：

```txt
ls       看文件
ls -a    看隐藏文件
ls -l    看详细信息
ls -la   看所有文件的详细信息
```

---

### 23.5 `cat` 是什么意思？

`cat` 是查看文件内容的命令。

比如：

```bash
cat README.md
```

意思是把 `README.md` 的内容直接打印到终端里。

它适合看短文件。  
如果文件很长，`cat` 会一下子刷很多内容，这时更适合用：

```bash
less README.md
```

或者只看前面一部分：

```bash
sed -n '1,100p' README.md
```

需要注意：不要随便 `cat` 可能包含密钥的配置文件，例如：

```bash
cat ~/.hermes/.env
cat ~/.codex/auth.json
```

这些文件里可能有 API Key、Token、账号信息。

更安全的做法是只搜索某个字段：

```bash
grep "OPENAI_BASE_URL" ~/.hermes/.env
```

### 23.6 Bash、CMD、PowerShell 是什么关系？

它们都可以叫“命令行环境”，但不是同一个东西。

可以先这样理解：

```txt
Bash        Linux / macOS / WSL 常见命令行
CMD         Windows 传统命令行
PowerShell  Windows 更现代、更强大的命令行
```

#### Bash

你在 WSL / Ubuntu 里用的通常就是 Bash。

常见命令：

```bash
ls
cd
pwd
cat
grep
find
rm
cp
mv
chmod
sudo
```

适合：

```txt
Linux 学习
WSL
服务器
开源项目安装脚本
Node / Python / Git 开发环境
Hermes 这类偏 Linux 风格的工具
```

#### CMD

CMD 是 Windows 老式命令行。

常见命令：

```cmd
dir
cd
type
copy
move
del
```

适合：

```txt
运行老的 .bat 脚本
执行一些简单 Windows 命令
```

现在新手不用重点学 CMD，知道它是 Windows 老命令行就够了。

#### PowerShell

PowerShell 是 Windows 上更现代的命令行。

常见命令：

```powershell
Get-ChildItem
Set-Location
Get-Content
Remove-Item
Copy-Item
Move-Item
```

不过 PowerShell 也兼容一些简单别名：

```powershell
ls
cd
cat
```

适合：

```txt
Windows 系统管理
查看端口
杀进程
配置 Windows 环境变量
执行 Windows 侧脚本
```

比如查 3000 端口：

```powershell
netstat -ano | findstr :3000
```

杀进程：

```powershell
taskkill /PID 进程ID /F
```

### 23.8 新手怎么判断该用哪个终端？

如果教程里出现这些：

```bash
ls
cat
grep
chmod
sudo
curl ... | bash
~/.xxx
```

一般是在 Bash / Linux / WSL 里执行。

如果教程里出现这些：

```powershell
Get-ChildItem
Set-ExecutionPolicy
taskkill
netstat -ano
$env:USERPROFILE
```

一般是在 PowerShell 里执行。

如果教程里出现这些：

```cmd
dir
type
del
copy
.bat
```

一般是在 CMD 里执行。

简单建议：

```txt
学 Linux / WSL / Hermes / 服务器：
优先用 Bash

处理 Windows 端口、进程、系统路径：
优先用 PowerShell

遇到老教程或 .bat 文件：
可能会用 CMD
```
