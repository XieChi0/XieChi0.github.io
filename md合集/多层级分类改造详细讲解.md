# 多层级分类系统改造讲解

> 本文档详细记录了如何将博客的分类系统从"两级"改造为"支持任意多层级"。从项目架构总览、文件职责划分，到每一行代码的逐行讲解，兼顾基础概念。

---

## 一、总述：改造目标与涉及范围

### 1.1 改造目标

原来博客的分类只支持两级，例如：
- `"编程"` —— 顶级分类
- `"编程/算法"` —— 顶级下的子分类

中间的层级会被忽略，只取首尾两段。改造后支持**任意多层级**，例如：
- `"编程/前端/框架/React"` —— 三级
- `"生活/旅行/2024/日本"` —— 四级

### 1.2 改造涉及的文件

| 文件路径 | 作用 | 是否新建 |
|---|---|---|
| `src/utils/content-utils.ts` | 数据层：把文章分类路径解析成树形结构 | 修改 |
| `src/components/widget/Categories.astro` | 侧边栏渲染：显示分类树 | 修改 |
| `src/components/CascadeSelector.svelte` | 导航栏级联选择器：hover 时展开多层级面板 | 修改 |
| `src/components/Navbar.astro` | 导航栏：把分类树数据转成级联选择器格式 | 修改 |
| `src/types/config.ts` | 类型定义：定义分类节点的数据结构 | 查看 |
| `src/config.ts` | 配置：定义导航栏显示哪些链接 | 查看 |

### 1.3 分类数据从文章到页面的流转路径


#### 3.1.3 核心算法：buildCategoryTree（这是本次改造的核心）

##### 改造前的问题

改造前的 `buildCategoryTree` 只能处理两级分类：

```typescript
// 改造前的问题代码

const childName = parts[parts.length - 1];  // 取最后一段："React"
```

如果分类是 `"编程/前端/React"`：
- 只取到了 `"编程"` 和 `"React"`
- 中间的 `"前端"` 被完全忽略了

##### 改造后的核心：递归插入算法

改造思路：**不预先假设层级数量，而是逐级往下走，每遇到一段就往树的下一层插入。**


> **函数参数类型解读：**
> `posts: { data: { category: string | null } }[]`
> - `posts` 是一个**数组**（最后的 `[]`）
> - 数组里每个元素的类型是 `{ data: { category: string | null } }`
> - `string | null` 表示类型可以是字符串，也可以是 null
> - 这是 Astro 内容集合的标准格式，`posts` 里面装的就是所有博客文章

```typescript

    const leafCounts: { [key: string]: number } = {};

    posts.forEach(post => {
        if (!post.data.category) {

            const ucKey = i18n(I18nKey.uncategorized);

        // .trim() 去前后空格
```

> **什么是"递归"？**
> 递归就是函数调用自己。想象你有一个人要爬一棵树，他爬到第一个分叉口，记录名字，然后继续往下爬到下一个分叉口，再记录名字...直到爬到叶子。
>
> `insertNodeRecursive` 做的事情就是：传入一个节点和剩余路径，函数把第一段插进去，然后**调用自己**处理剩余的路径段。

```typescript

        // 这是一个"递归终止条件"，没有它函数会无限循环
```

```typescript

        //      ↑ ES6 解构赋值：取数组第一个元素
        //                     ↑ ↑ 把剩余元素组成新数组

        //   currentName = "前端"

        //   currentFullPath = "编程/前端"
```

> **`${}` 在模板字符串中的作用：**
> ```typescript
> const name = "小明";
> const greeting = `你好，${name}！`;  // "你好，小明！"
> ```
> 用反引号 `` ` `` 而不是引号 `'`，里面用 `${变量}` 插入值。

> **`find` vs `filter` 的区别：**
> - `arr.filter(fn)`：返回**所有**让函数返回 true 的元素组成的数组
> - `arr.find(fn)`：返回**第一个**让函数返回 true 的元素，只有一个

> **`!node` vs `node === undefined` 的区别：**
> - `node === undefined` 是严格比较，只有值是 `undefined` 才为 true
> - `!node` 是隐式转换，只要值是 falsy（`undefined`、`null`、`0`、`""`、`false`）就为 true
> - 但 `find()` 找不到时返回 `undefined`，所以两者在这里效果相同
> - 代码里两种写法都会出现，都是正确的

##### 计算父节点总数

```typescript
    // 第 182-189 行：计算每个父节点的文章总数
    function computeParentCount(node: CategoryNode) {
        // 用后序遍历：先处理所有子节点，再处理自己
        if (node.children && node.children.length > 0) {
            node.children.forEach(computeParentCount);
            // 递归先处理子节点

            node.count += node.children.reduce((sum, child) => sum + child.count, 0);
            // .reduce() 是数组方法，把数组"归约"成一个值
            // 从 0 开始，把每个 child.count 加起来
        }
    }
    root.forEach(computeParentCount);
```

> **`.reduce((累加器, 当前元素) => 累加器 + 当前元素, 初始值)` 详解：**
> ```typescript
> [1, 2, 3].reduce((sum, n) => sum + n, 0)
> // 第一次：sum=0, n=1 → 返回 1
> // 第二次：sum=1, n=2 → 返回 3
> // 第三次：sum=3, n=3 → 返回 6
> // 最终结果：6
> ```
>
> 这段代码把 `children` 数组里所有 `count` 加起来，累加到父节点的 `count` 上。

##### 排序

```typescript
    // 第 191-200 行：对所有层级的节点按字母排序
    function sortNodes(nodes: CategoryNode[]) {
        nodes.sort((a, b) => a.name.toLowerCase().localeCompare(b.name.toLowerCase()));
        // .sort() 是数组方法，会直接修改原数组
        // 传入比较函数：(a, b) => 返回负数 a 在前，返回正数 b 在前
        // .toLowerCase() 转小写，避免 "Apple" 排在 "banana" 前面
        // .localeCompare() 按语言习惯比较字符串（考虑中文、特殊字符等）

        nodes.forEach(node => {
            if (node.children && node.children.length > 0) {
                sortNodes(node.children);  // 递归排序子节点
            }
        });
    }
    sortNodes(root);
```

> **为什么排序要用 `.localeCompare` 而不是直接 `a.name < b.name`？**
> - `a.name < b.name` 用的是 Unicode 码点比较，中文会按 Unicode 顺序排
> - `.localeCompare()` 会根据当前语言环境排序，中文会按拼音顺序
> - 对于中文分类名（如"编程"、"生活"），localeCompare 会按拼音首字母排序

#### 3.1.4 递归展平函数 flatten（改造后）

```typescript

export async function getCategoryList(): Promise<Category[]> {
    const tree = await getCategoryTree();

    function flatten(nodes: CategoryNode[]): Category[] {
        const result: Category[] = [];

        function walk(node: CategoryNode) {
            // 先把自己加入结果
            result.push({
                name: node.name,
                count: node.count,
                url: node.url,
            });
            // 然后递归处理所有子节点
            if (node.children) {
                node.children.forEach(walk);
            }
        }

        nodes.forEach(walk);
        return result;
    }

    return flatten(tree).sort((a, b) =>
        a.name.toLowerCase().localeCompare(b.name.toLowerCase())
    );
}
```

> **什么是"展平"（flatten）？**
> 把树形结构转成一维数组：
> ```
> 树:  编程
>       └── 前端
>             └── React
>
> 展平后: [编程, 前端, React]
> ```
>
> 深度优先遍历（DFS），先父后子再兄弟。

#### 3.2.3 递归渲染函数（核心改造）

```typescript
// 第 24-54 行：递归渲染分类树，返回 HTML 字符串
// indent 是缩进深度，初始为 0，每深入一层加 1
function renderCategoryNodes(nodes: CategoryNode[], indent: number = 0): string {
    // .map() 遍历每个节点，生成对应的 HTML 片段，然后用 .join('') 拼接成一个大字符串
    return nodes.map(node => {
        const paddingLeft = `${0.75 + indent * 1.25}rem`;
        // 缩进计算：
        //   第 0 层：paddingLeft = 0.75 + 0 * 1.25 = 0.75rem
        //   第 1 层：paddingLeft = 0.75 + 1 * 1.25 = 2.0rem
        //   第 2 层：paddingLeft = 0.75 + 2 * 1.25 = 3.25rem

        const hasChildren = node.children && node.children.length > 0;
        // 有子节点 → 用 <details> 折叠显示
        // 无子节点 → 用普通 <a> 链接

        if (hasChildren) {
            // 有子分类：用 <details> 折叠元素
            return `
                <details class="category-parent" open>
                    <summary class="category-summary" style="padding-left: ${paddingLeft}">
                        <a href="${node.url}" class="category-link">${node.name.trim()}</a>
                        <span class="category-count">${node.count}</span>
                    </summary>
                    <div class="category-children">
                        ${renderCategoryNodes(node.children, indent + 1)}
                        <!-- ↑ 递归调用！indent + 1 递进到下一层 -->
                    </div>
                </details>
            `;
        } else {
            // 叶子节点：普通链接样式
            return `
                <a href="${node.url}" class="category-leaf" style="padding-left: ${paddingLeft}">
                    <span class="leaf-name">${node.name.trim()}</span>
                    <span class="category-count">${node.count}</span>
                </a>
            `;
        }
    }).join('');
    // .map() 返回数组，.join('') 把数组合并成字符串
}
```

> **`${变量}` 在模板字符串中的作用：**
> ```typescript
> const name = "小明";
> const html = `<span>${name}</span>`;  // "<span>小明</span>"
> ```
> 模板字符串里面可以嵌入任何 JS 表达式，如 `${paddingLeft}`、`${node.count}`、`${renderCategoryNodes(...)}`

> **为什么用 `set:html` 而不是直接 `<Component>`？**
> `renderCategoryNodes` 返回的是一个**字符串**（HTML 代码文本），不是真实的 DOM 元素。
> `set:html` 是 Astro 的特殊指令，作用是："把这个字符串作为 HTML 代码直接插入到 DOM 里"。

#### 3.3.3 核心派生状态

```svelte
// 第 23-33 行：用 $derived.by 定义派生状态（响应式自动计算）
let menus = $derived.by(() => {
    let opts = options;      // opts 是当前列的数据
    const result = [opts];   // result 收集所有列，第一列是根级

    for (const idx of activePath) {
        // activePath 如 [0, 1] 表示：第 0 列选第 0 项，第 1 列选第 1 项
        if (opts[idx]?.children) {
            // opts[idx] 是当前选中的项
            // opts[idx]?.children 是可选链，如果 children 不存在返回 undefined
            opts = opts[idx].children!;  // !是非空断言，表示"我确定 children 存在"
            result.push(opts);           // 把下一列加入结果
        }
    }
    return result;  // 返回所有列的数组，列数 = activePath.length + 1
});
```

> **什么是 `$derived`？**
> `$derived` 是 Svelte 5 的新响应式 API，相当于 Svelte 4 的 `$:` 推导值。
> - `$derived(expr)`：把表达式的结果变成响应式状态
> - `$derived.by(() => {...})`：当函数内部依赖的状态变化时，自动重新计算

> **什么是 `!` 非空断言？**
> `opts[idx].children!` 中的 `!` 告诉 TypeScript："我保证这个值不是 null 或 undefined，你不用检查了"。
> 这在确认有子节点时才执行这个分支，所以可以安全断言。

#### 3.3.4 交互逻辑

```svelte
// 第 63-71 行：鼠标悬停时的处理
function hoverItem(colIdx: number, itemIdx: number) {
    const item = menus[colIdx][itemIdx];
    // 获取当前 hover 的项

    activePath = activePath.slice(0, colIdx);
    // 把 activePath 截断到当前列
    // 例如 activePath = [0, 1, 2]，hover 第 0 列 → 截断为 []
    // 例如 activePath = [0, 1, 2]，hover 第 1 列 → 截断为 [0]

    if (item.children && item.children.length > 0) {
        // 如果 hover 的项有子节点，把它的索引加入 activePath
        activePath = [...activePath, itemIdx];
        //      ↑ 展开运算符，把数组展开后创建新数组
        //  等价于 activePath = activePath.concat([itemIdx])
    }
    // menus 会自动重新计算，因为 activePath 变了
}
```

> **`.slice(0, colIdx)` 截断逻辑：**
> ```typescript
> activePath = [0, 1, 2]  // 表示：第0列选0，第1列选1，第2列选2
>
> hoverItem(0, 0)  // colIdx=0
> activePath = activePath.slice(0, 0)  // → []（从0截到0，不含index 0）
>
> hoverItem(1, 1)  // colIdx=1
> activePath = activePath.slice(0, 1)  // → [0]（保留index 0）
>
> hoverItem(2, 0)  // colIdx=2
> activePath = activePath.slice(0, 2)  // → [0, 1]（保留index 0,1）
> ```
>
> 这样 hover 从深层回到浅层时，会正确截断 activePath，不会保留深层的信息。

```astro
    return tree.map(node => {
        const fullPath = parentPath ? `${parentPath}/${node.name}` : node.name;
```

## 四、总结

### 4.1 改造的核心思想

本次改造围绕一个核心问题：**如何让程序自动处理任意层级的分类路径？**

答案是**递归**。无论是数据构建还是 UI 渲染，都用递归代替了原来固定两层的循环。

### 4.2 涉及文件的职责分层

```
展示层（用户看到的）
    ├── Categories.astro     → 侧边栏：递归渲染 HTML 字符串
    └── CascadeSelector.svelte → 导航栏：递归渲染面板列

数据转换层（承上启下）

数据构建层（幕后英雄）

```
