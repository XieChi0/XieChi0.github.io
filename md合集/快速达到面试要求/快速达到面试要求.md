- state 怎么用

- 列表渲染

- useEffect

# **建议你写的 React 项目**

写一个：

> React 博客知识库检索面板

功能非常小，但很适合你：

- 左侧文章列表
- 顶部搜索框
- 标签/分类筛选
- 右侧文章详情
- 收藏/最近阅读
- 一个 ECharts 图表：分类文章数量
- 一个“AI 问答”输入框，先假数据，后面接 RAG

技术栈：

- React + Vite + TypeScript
- React Router
- Zustand 或直接 useState
- Axios/fetch
- ECharts
- Tailwind 或普通 CSS

这比 Todo 高级，又不会太大。更重要的是，它能和你的博客内容绑定，面试时很好讲。

# **React 两天冲刺安排**

第 1 天上午：

- 看 JSX、组件、props、state。
- 把 Vue 的思维迁移过来：
  - Vue 的 data/ref ≈ React 的 useState
  - Vue 的 computed ≈ React 的 useMemo
  - Vue 的 watch/onMounted ≈ React 的 useEffect
  - Vue 的 v-for ≈ JS 的 map
  - Vue 的 v-if ≈ 条件表达式

第 1 天下午：

- 写项目壳子。
- 完成文章列表、搜索、分类筛选、详情切换。
- 熟悉 useState、props、组件拆分。

第 2 天上午：

- 加 useEffect 模拟接口请求。
- 加 React Router。
- 加一个自定义 Hook：usePosts。
- 加一个 ECharts 分类图。

第 2 天下午：

- 准备面试问答。
- 写 README。
- 总结 React vs Vue。
- 把项目截图、功能点、技术点整理成简历话术。

# **RAG / LangChain 怎么上手**

先别一上来就 LangChain 全家桶。最快理解路线是：

RAG 本质就一句话：

> 先从资料库里找相关内容，再把相关内容塞给大模型回答。

它分 5 步：

1. 读取文档：你的 Markdown 博客。
2. 切块：把文章切成小段。
3. 向量化：把文本变成 embedding。
4. 检索：用户提问时找最相关的段落。
5. 生成：把检索到的内容和问题一起交给模型。

LangChain 是什么？

> LangChain 是把“读文档、切块、向量库、检索器、模型调用、链式流程”封装起来的工具库。

你面试最低要能讲：

- RAG 解决大模型不知道私有知识的问题。
- Embedding 是把文本转成向量，方便相似度检索。
- 向量数据库可以用 Chroma、FAISS、Qdrant。
- LangChain 可以帮你组织 RAG 流程，但不是必须的。
- RAG 的关键问题是切块、召回质量、上下文长度、幻觉控制。

# **RAG 最小 Demo**

你做这个就够：

> 基于本地 Markdown 博客的问答系统

最小版本可以不用 LangChain：

- Node 或 Python 读取 src/content/posts
- 按标题和段落切块
- 先做关键词检索
- 输入问题
- 返回相关文章片段
- 再接大模型总结

然后升级版再说：

- 用 LangChain 的 Document Loader
- Text Splitter 切块
- Embedding
- Chroma/FAISS
- Retriever
- QA Chain

如果时间特别紧，先做关键词检索版，也能讲成：

> 我先实现了一个轻量 RAG 原型，检索部分目前用关键词匹配，后续可以替换成 embedding + 向量数据库。核心链路是一样的。

# **你现在最优顺序**

我建议：

第 1 天：JS + Vue 面试冲刺
重点：闭包、原型、事件循环、Promise、Vue 响应式、组件通信、路由、Pinia/Vuex、性能优化。

第 2-3 天：React 小项目
目标：能写基础 React 页面，能讲 Hooks 和 React/Vue 区别。

第 4 天：RAG 最小 Demo
目标：能把你的博客 Markdown 检索出来，形成“知识库问答”的雏形。

第 5 天：LangChain 补概念 + 小改造
目标：知道 LangChain 在 RAG 里每一步对应什么模块。

**你是不是应该写 React 项目？**
是，应该写。

但项目目标要克制：**只做能证明 React 基础能力的项目，不做完整业务系统。**

最推荐题目：

> React + TypeScript 博客知识库检索面板

它刚好一箭三雕：

- 证明你了解 React。
- 接上你的博客内容。
- 后续可以自然扩展成 RAG / AI 问答。

这条线最快，也最像一个聪明的面试准备方案。
