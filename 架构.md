### 4. 组件结构(`src/components/`):

**Astro 组件**

- `Layout.astro` - 基础 HTML 布局（SEO、主题脚本、PhotoSwipe）
- `MainGridLayout.astro` - 主网格布局（导航栏、Banner、内容、侧边栏、Footer）
- `Navbar.astro` - 顶部导航栏
- `Footer.astro` - 页脚
- `PostPage.astro` - 文章卡片列表
- `PostCard.astro` - 单篇文章卡片
- `PostMeta.astro` - 文章元数据（日期、标签、分类）

- `Search.svelte` - 全文搜索（集成 Pagefind）
- `LightDarkSwitch.svelte` - 明暗模式切换
- `DisplaySettings.svelte` - 显示设置面板
- `ArchivePanel.svelte` - 归档面板

**小部件** (`src/components/widget/`):

- `Profile.astro` - 作者信息卡片
- `Categories.astro` - 分类列表
- `Tags.astro` - 标签云
- `TOC.astro` - 目录导航
- `SideBar.astro` - 侧边栏容器

### 5.布局层级

在页面结构上好几个地方都有表示布局的含义，在这里区分一下它们的含义。

Layout.astro      = 毛坯房框架（地基、墙体、屋顶）

MainGridLayout    = 装修方案（客厅、卧室、厨房在哪）

Components        = 家具电器（沙发、床、冰箱）

1️⃣src/layout/layout.astro ---HTML骨架层

职责：定义页面HTML结构底层逻辑

```
---
// ✅ 做的事情：
// 1. 定义 <html>, <head>, <body> 标签
// 2. 加载字体 (Roboto)
// 3. 设置 <title>, <meta> SEO 标签
// 4. 生成 favicon 链接
// 5. 初始化主题 (暗色/亮色) 的脚本
// 6. 加载 OverlayScrollbars 滚动条
// 7. 配置 PhotoSwipe 图片灯箱
// 8. 注册 Swup 页面过渡的 hooks
// 9. 滚动时隐藏/显示导航栏的逻辑
---

<!DOCTYPE html>
<html lang={siteLang}>
  <head>
    <title>{pageTitle}</title>
    <meta charset="UTF-8" />
    <meta name="description" content={...}>
    <!-- ... 更多 meta 标签 ... -->
  </head>
  <body>
    <ConfigCarrier></ConfigCarrier>
    <slot />  <!-- 页面内容插槽 -->
  </body>
</html>
```

2️⃣ src/layouts/MainGridLayout.astro   ---页面结构层  
职责：定义页面的视觉布局结构（Banner+网格+侧边栏）

```
---
// src/layouts/MainGridLayout.astro

// ✅ 做的事情：
// 1. 导入 Layout 作为基础
// 2. 定义 Banner 区域
// 3. 定义主网格 (grid) 布局
// 4. 放置 Navbar、SideBar、内容区、Footer
// 5. 设置响应式列数
---

<Layout ...>
  <!-- 导航栏 -->
  <div id="top-row">
    <Navbar></Navbar>
  </div>

  <!-- Banner 大图 -->
  <div id="banner-wrapper">
    <ImageWrapper ... />
  </div>

  <!-- 主网格内容 -->
  <div class="grid grid-cols-[17.5rem_auto] ...">
    <SideBar ... />     <!-- 侧边栏 -->
    <main>              <!-- 文章内容 -->
      <slot></slot>
    </main>
  </div>

  <Footer></Footer>     <!-- 页脚 -->
</Layout>
```

**核心区别**


| **对比**   | `Layout.astro`               | `MainGridLayout.astro` |
| -------- | ---------------------------- | ---------------------- |
| **层级**   | HTML 文档根                     | 页面视觉结构                 |
| **职责**   | 底层框架                         | 上层布局                   |
| **是否通用** | 所有页面共享                       | 大部分页面共享                |
| **内容**   | `<html>`, `<head>`, `<body>` | Banner、Grid、Sidebar    |


3️⃣src/components/*.astro 

可复用的UI组件


| **组件**           | **职责**                          |

| `Navbar.astro`   | 顶部导航：Logo、链接、搜索按钮、主题切换          |

| `TOC.astro`      | 文章目录                            |

### 7. 内容管理方式

**内容集合**: Astro Content Collections (`src/content/`)

**集合配置** (`src/content/config.ts`):

```typescript
schema: {
  title: string           // 文章标题
  published: Date         // 发布日期
  updated?: Date          // 更新日期
  draft?: boolean         // 草稿状态
  description?: string    // 描述
  image?: string          // 封面图
  tags?: string[]         // 标签
  category?: string       // 分类
  lang?: string           // 语言
  // 内部使用
  prevTitle/prevSlug      // 上一篇
  nextTitle/nextSlug      // 下一篇
}
```

**文章目录**: `src/content/posts/`

- 现有示例文章: `markdown.md`, `expressive-code.md`, `video.md`, `draft.md`, `markdown-extended.md`, `guide/index.md`

### 9. 构建和部署配置

**npm 脚本**:

```json
{
  "dev": "astro dev",           // 开发服务器
  "build": "astro build && pagefind --site dist",  // 构建 + 搜索索引
  "preview": "astro preview",   // 预览
  "check": "astro check",       // 类型检查
  "type-check": "tsc --noEmit", // TypeScript 检查
  "format": "biome format --write",  // 格式化
  "lint": "biome check --write"      // 代码检查
}
```

**GitHub Actions** (`.github/workflows/build.yml`):

- Node.js 22 和 23 双重测试
- Astro Check 类型检查
- Astro Build 构建验证

**搜索配置** (`pagefind.yml`):

- 排除 KaTeX 公式
- 排除搜索面板
- 忽略标记元素

**部署目标**: Vercel (`https://fuwari.vercel.app/`)
