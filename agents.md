# 开发规范 (AI)

## 环境要求

- Node.js: **22.x 或 23.x**（必须与 CI 一致）
- pnpm: 最新版本

## 推送前检查

按顺序执行，全部通过后再推送：

```bash
pnpm biome ci ./src
pnpm astro check
pnpm astro build
```

## 常见问题

| 问题 | 解决方案 |
|-----|---------|
| TypeScript `undefined` 错误 | 用 `?.` 或 `??` 处理，或加空检查 |
| `any` 类型警告 | 定义具体 interface |
| Biome 格式错误 | `pnpm biome check --write ./src` |
| 导入排序错误 | 同上，自动修复 |

## 代码规范

- 优先使用 `?.` 和 `??`，避免非空断言 `!`
- 用模板字符串 `` `${var}` ``，不用拼接
- 导入按字母排序
- 避免 `@ts-ignore`

## 用户情况

当前用户在**面试准备阶段**，很多技术点没有实际项目经验。

### 对 Agent 的期望

1. 讲概念时，**顺便带代码示例**（最简单的实现方式）
2. 提到用到的库或框架，**给出名字**
3. 不要过度深入，知道**怎么用、解决什么问题**就够了
4. 如果有现成的工具或库，优先告诉用户，**不要让用户自己造轮子**

## 触发模式

### [讲解]

1. **先用大白话解释**
   - 先说明这个东西是干什么的、可以用来做什么。
   - 然后可以说说专业词、专业术语。

2. **说明它解决什么问题**
   - 讲清楚为什么需要它。
   - 它解决了开发中的什么麻烦。

3. **讲整体用法**
   - 先给整体流程或使用思路。
   - 再进入具体代码，不要直接丢复杂代码。

4. **代码要分步骤讲**
   - 代码尽量拆成小段。
   - 每段都说明：为什么这么写、表达什么意思、起什么作用。

5. **顺带解释相关语法**
   - 如果涉及 TypeScript、Vue、JavaScript、Nest、Express、Prisma 等语法，要顺手解释。
   - 不要默认用户已经懂。

6. **涉及设计思路时，用简单例子说明**
   - 说明它的好处和适用场景。
   - 避免只讲抽象概念。

7. **控制复杂度**
   - 不要一次性给特别长、特别难懂的代码。
   - 优先使用“小例子 + 分步骤解释”。

8. **最后给记忆口诀**
   - 用一句容易记的话总结核心理解。
   - 方便之后快速回忆。

### [提交总结]

当你在聊天中输入 `[提交总结]` 时，执行以下操作：

1. **收集本次会话的变更信息**：
   - 使用 `git diff` 查看已修改的文件
   - 分析每个文件的变更目的

2. **生成结构化总结**，输出到 `docs/commit-history.md`：

```markdown
## [时间] 提交摘要

**Commit ID**: (提交后补充)
**变更文件数**: N
**主要需求**: 简要描述

### 变更详情

| 文件 | 变更类型 | 说明 |
|-----|---------|------|
| src/xxx.ts | 修改 | xxx |
| src/yyy.astro | 新增 | xxx |

### 实现方式

- xxx
- yyy

---
```

1. **追加到 `docs/commit-history.md` 底部**（文件不存在时自动创建）

### 注意事项

- 如果某些信息在提交前无法确定（如 commit ID），留空并在括号中注明
- commit ID 在 `git commit` 执行后补充到对应条目
- 保持格式一致，便于后续查阅

### [everydaylearn]

当你在聊天中输入 `[everydaylearn]` 时，执行以下操作：

1. **新建当天日期的 MD 文件**：
   - 文件路径：`src/content/posts/everyDayLearn/0515.md`（日期格式：MMDD）
   - 格式参考同目录下已有的 `0429.md`

2. **内容来源**：
   - 用户会告诉你本次要总结哪些内容
   - 你只需要把用户指定的每个主题作为一个一级标题写入文件
   - 每个主题下的具体内容，由你根据本次讨论填写

3. **自动生成 tags**：
   - 根据每个大标题（一级标题 `#`）的主题，提取关键知识点作为 tags
   - 每个标题提取 1-2 个核心关键词（如"对象字面量"、"相等比较"、"异常处理"）
   - 不要使用太大范围的标签（如 "JavaScript"、"前端"），要用具体的技术点
   - tags 数组放在 YAML frontmatter 的 `tags` 字段中

### [yaml]

当你在聊天中输入 `[yaml]` 时，执行以下操作：

1. **为当前打开的 MD 文件添加 YAML frontmatter**：
   - 在文件顶部添加标准的 YAML frontmatter
   - 使用以下模板：

```yaml
---
title: (根据文章标题填写)
published: (根据实际情况填写，格式如 2024-05-01)
updated: (根据实际情况填写，格式如 2024-11-29)
description: '(简短描述文章内容)'
image: ''
tags: [(根据文章主题填写，如 [教程, 前端])
category: 'everyDayLearn'
draft: false
---
```

1. **具体值根据文章内容确定**：
   - `title`：使用文件名或文章主题
   - `published`：首次发布日期
   - `updated`：最后更新日期
   - `description`：1-2 句话概括文章
   - `tags`：与文章相关的标签数组
   - `category`：所属分类
   - `draft`：默认为 `false`，草稿则为 `true`
