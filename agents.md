# 开发规范 (AI)

## 环境要求

- Node.js: **22.x 或 23.x**（必须与 CI 一致）
- pnpm: 最新版本

## 推送前检查

按顺序执行，全部通过后再推送：

```bash
pnpm biome ci ./src
pnpm astro check
pnpm astro build
```

## 常见问题

| 问题 | 解决方案 |
|-----|---------|
| TypeScript `undefined` 错误 | 用 `?.` 或 `??` 处理，或加空检查 |
| `any` 类型警告 | 定义具体 interface |
| Biome 格式错误 | `pnpm biome check --write ./src` |
| 导入排序错误 | 同上，自动修复 |

## 代码规范

- 优先使用 `?.` 和 `??`，避免非空断言 `!`
- 用模板字符串 `` `${var}` ``，不用拼接
- 导入按字母排序
- 避免 `@ts-ignore`

## 触发模式

### [科普]

当你在聊天中输入 `[科普]` 时：
- 需要先用通俗的方式介绍这个东西是干什么的、总体用法是什么
- 在讲用法时，顺带讲讲相关语法（比如 TypeScript / Vue / JavaScript 等）
- 如果涉及到某种设计思路或设计模式，要用简单例子说明它的好处、适用场景
- 尽量避免一次性给特别大段、难懂的代码，分步骤给并配简单说明

### [提交总结]

当你在聊天中输入 `[提交总结]` 时，执行以下操作：

1. **收集本次会话的变更信息**：
   - 使用 `git diff` 查看已修改的文件
   - 分析每个文件的变更目的

2. **生成结构化总结**，输出到 `docs/commit-history.md`：

```markdown
## [时间] 提交摘要

**Commit ID**: (提交后补充)
**变更文件数**: N
**主要需求**: 简要描述

### 变更详情

| 文件 | 变更类型 | 说明 |
|-----|---------|------|
| src/xxx.ts | 修改 | xxx |
| src/yyy.astro | 新增 | xxx |

### 实现方式

- xxx
- yyy

---
```

3. **追加到 `docs/commit-history.md` 底部**（文件不存在时自动创建）

### 注意事项

- 如果某些信息在提交前无法确定（如 commit ID），留空并在括号中注明
- commit ID 在 `git commit` 执行后补充到对应条目
- 保持格式一致，便于后续查阅

### [everydaylearn]

当你在聊天中输入 `[everydaylearn]` 时，执行以下操作：


1. **新建当天日期的 MD 文件**：
   - 文件路径：`src/content/posts/everyDayLearn/0515.md`（日期格式：MMDD）
   - 格式参考同目录下已有的 `0429.md`


2. **内容来源**：
   - 用户会告诉你本次要总结哪些内容
   - 你只需要把用户指定的每个主题作为一个一级标题写入文件
   - 每个主题下的具体内容，由你根据本次讨论填写
